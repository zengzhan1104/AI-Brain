# ingest 技能

## 全局约束

**禁止为进度或上下文压力牺牲单篇阅读深度。** 每篇论文的阅读质量是唯一优先级——不考虑"会话还剩下多少空间"、"还有多少篇没处理"、"如何在本次对话中多完成几篇"。如果在处理完当前这篇之前上下文即将耗尽，应在该篇处理完毕后自然暂停，下次会话继续。

**禁止简化模板填充。** 模板的所有区域（摘要/背景/方法或叙事逻辑/结论或批判性评论/个人判断/关联连接等）必须在每篇论文中完整填充。研究纲领比对已后移至模板主体填充完成之后（步骤 3c），因此填充 3a 时尚未知晓关联度高低，不存在因"关联度低"而选择性跳过的问题。

---

## 核心工作流：Inbox & Archive

你正在维护一个 **LLM Wiki**（Obsidian 知识库）。`raw/` 目录是"待处理文件库"，`wiki/` 是"编译输出层"。

**目录结构约定：**
- `raw/papers/` — 待处理文件（Markdown）
- `raw/archive/` — **已处理文件的归档目录，禁止读取**
- `wiki/sources/` — 论文精读
- `wiki/concepts/` — 概念（框架、方法论、理论、专业名词）
- `wiki/methods/` — 测试、表征与数据处理方法

---

## 触发逻辑

1. **用户执行 `/ingest`**：扫描 `raw/` 所有子目录（排除 `archive/`），找出待处理文件。按文件名排序后，逐篇依次处理（详见「单篇确认规则」）。
2. **用户执行 `/ingest <path>`**：仅处理指定文件（若为目录，按文件名排序后逐篇处理）。
3. **隐式触发**：用户说"把这个资料摄入知识库"、"导入这篇文章"时，自动执行 ingest。

---

## 单篇确认规则

**取消批次制。** 每篇论文独立处理，没有"批"的概念。处理完一篇后立即暂停，报告该篇完成情况，等待用户确认后再处理下一篇。

- **逐篇确认**：每篇论文完成步骤 1–7 后，输出单篇完成摘要并等待用户确认。
- **质量优先于会话连续性**：禁止为了在单次会话中完成更多论文而压缩单篇阅读深度。若会话因长度限制难以维持深度，应在当前论文处理完毕后自然暂停，下次会话继续。每篇论文必须完整阅读全文后再填充模板字段，不得仅读摘要替代。
- **严禁并行 Agent**：**绝对禁止**使用 Agent 工具将不同论文分发给子 agent 并行处理。所有论文必须在主会话中逐篇顺序处理。
- **强制阅读确认**：每篇论文在步骤 1 读取完成且所有章节标注 ✓ 之后，方可进入步骤 2。此清单应直接输出给用户，不做隐藏或折叠。清单格式及规则见步骤 1。

## 编译流水线

对每个待处理源文件，严格按以下步骤执行：

### 步骤 1：读取源文件

#### 步骤 1a：前置声明（在读取正文之前执行）

**先用 Grep 提取论文的章节结构**（搜索 `^# ` 和 `^## `），输出一份"预读声明"——

```
## 预读声明：{论文标题}

| 序号 | 章节 | 状态 |
|------|------|------|
| 1 | Abstract | 待读 |
| 2 | §1 Introduction | 待读 |
| 3 | §2 ... | 待读 |
| ... | （全部一/二级章节逐行列出） | |
| N | §N Conclusions | 待读 |

我将在阅读每个章节后逐一确认，全部标注为 ✓ 后方可进入步骤 2。
```

此声明的目的是**在阅读开始前就锁定论文的完整章节清单**，防止在阅读过程中选择性跳过某章。

#### 步骤 1b：逐章阅读正文

- **如果是 `.md` 文件**：使用读取工具完整读取内容。若文件超过读取限制，先读标题+摘要+目录（首部），再逐章分段读取正文各章节，最后读结论部分。**注意**：分段读取时，每次读取的末尾与下一次读取的开头必须有内容重叠（约 50 行），确保跨越读取边界的段落不会丢失。
- **若论文目录下存在支撑材料 Markdown**：当论文位于子目录模式（`{年份 - 标题}/`）下时，检查目录中是否存在 `{年份} - {标题} - 支撑材料.md`、`{年份} - {标题} - 支撑材料-2.md` 等匹配 `{年份} - {标题} - 支撑材料*.md` 的文件。若存在，一并读取。支撑材料内容在主文献内容之后按顺序处理，其提炼内容写入精读笔记的 `## 支撑材料` 区域。
- **若论文目录下存在 `_zotero_meta.json`**：读取其中的 `zotero_item_id` 和 `zotero_collections`，存入内部变量 `{zotero_item_id}` 和 `{zotero_collections}`，供步骤 3 填充模板 frontmatter 使用。若文件不存在，两个变量均设为空字符串。

#### 步骤 1c：输出阅读确认清单

当所有章节阅读完成后，将步骤 1a 的预读声明表更新为"阅读确认清单"，每个章节标注 ✓/✗ 和"核心内容一句话"：

  | 章节 | 已读 | 核心内容一句话 |
  |------|------|----------------|
  | Abstract / 摘要 | ✓/✗ | ... |
  | §1 Introduction / 引言 | ✓/✗ | ... |
  | §2 ... | ✓/✗ | ... |
  | （按论文实际章节逐行列出，直至结论） | | |
  | §N Conclusions / 结论 | ✓/✗ | ... |

  清单规则：
  - 每一行必须是论文**实际存在的一级或二级章节标题**，不得合并多章为一行、不得省略任何章节。
  - "核心内容一句话"必须有**该章节的具体信息**（一个关键数字、一个核心论断、一个方法论细节），禁止写"介绍了XX"、"综述了XX"等泛泛描述。
  - 若某章节因文件超出读取长度限制而未被读到，**必须**在"已读"列标注 ✗，然后继续读取剩余章节，直到所有章节均标注 ✓。
  - **所有章节确认 ✓ 后，方可进入步骤 2。** 若任一章存在 ✗，必须补读该章节。

### 步骤 2：提炼核心并翻译

从源文件中提取：
- **核心主旨**：这段资料讲什么（1-2句话）
- **概念**：值得建立独立页面的术语，按论文类型侧重不同：
  - **研究论文**：优先提取新材料、新物理现象、关键性能参数
  - **综述论文**：优先提取分类框架、技术路线/学派、领域基准指标、里程碑工作

**若源文件为研究论文（非综述），额外提取**：论文中的测试/表征/数据处理方法（名称、测量的物理量、关键实验条件、支撑的结论），填入精读笔记的「涉及的表征方法」区域。此提取不触发 `wiki/methods/` 页面创建——方法页面按需触发（见步骤 5）。

**额外执行综述检测**：

在提取核心主旨后，判断该论文是否为综述类文献。通过以下信号综合判定（满足任意 2 条即认定为综述）：

1. **标题关键词**：标题含 `review`、`survey`、`综述`、`进展`、`回顾`、`展望`、`overview`、`state-of-the-art`、`state of the art`、`critical review`、`literature review`、`bibliometric`、`bibliography`
2. **摘要/引言特征**：摘要首句或引言末段出现 "This paper reviews"、"We review"、"This survey"、"A systematic review"、"本文综述"、"本文回顾" 等综述标志句式
3. **文章类型标记**：论文首页或元数据中标明 "Review Article"、"Review Paper"、"Survey"、`documentType: Review`
4. **结构特征**：正文以分类框架（taxonomy / classification）为主线组织，引用文献数量显著偏大（通常 > 100 篇），且无独立的实验方法章节
5. **路径兜底**：若论文子目录名（即 `raw/papers/{年份 - 标题}/` 中的 `{标题}` 部分）包含 `综述` 或 `review`（不区分大小写），**强制执行** `{is_review_paper} = true`。此规则仅在内容信号不充分（4 条信号均不满足）时作为兜底，避免因上层分类目录名含"综述"而将其中所有论文误标为综述。

**检测结果标记**：将判定结果存储为内部变量 `{is_review_paper}`（`true` / `false`），供步骤 3 选择模板时使用。**内容信号优先**：路径兜底仅在 4 条内容信号均不满足时启用，内容信号判定的 `true` 优先于路径兜底。

精读笔记的撰写内容均使用中文。原文引用保留原语言，不在翻译范围内。

**提取到的数学公式和变量符号必须规范化**：
- 所有公式使用 Obsidian LaTeX 数学分隔符：行内 `$...$`，块级 `$$...$$`
- Markdown 中出现的 Unicode 呈现形式（`μ`、`β`、`·`、`Ω` 等）**必须转换**为 LaTeX 等价命令（`\mu`、`\beta`、`\cdot`、`\Omega` 等）
- 上下标使用 LaTeX 语法：`$R_{j-a}$`（非 `Rj-a`）、`$T_0$`（非 `T0`）、`$E_c^3$`（非 `Ec³`）
- 若 Markdown 中保留有原始 LaTeX 源码（如 `\nabla \cdot (k_{th} \nabla T)`），直接包裹在 `$$...$$` 中即可
- 表格内的公式同样需要 `$...$` 包裹

### 步骤 3：创建来源摘要

根据**综述检测结果**，选择对应模板并读取：

- `{is_review_paper}` = `false` → **模板 A**（`.claude/skills/ingest/研究论文精读模板.md`）
- `{is_review_paper}` = `true` → **模板 B**（`.claude/skills/ingest/综述论文精读模板.md`）

**子库名前缀**：模板中 `{{子库名}}` 占位符填入当前目录名（`basename "$PWD"`），使 `sources_main` / `sources_supplementary` 路径以子库目录名为前缀。

**保存路径与命名规范**：`wiki/sources/{年份} - {标题}.md`（格式为 `年份 - 论文标题.md`，中文论文用中文标题，英文论文保留英文标题）。

#### 步骤 3a：填充不依赖研究纲领的字段

读取对应模板文件后，按模板结构逐区域填充。模板中以 `<!-- ⤷ 3c 回填 -->` 标记的字段在 3a 中**留空**，其余所有字段均填入实质性内容。**禁止**在非标记字段写入"待填充""暂无"等占位符——3a 中填充的字段即应定稿。填充完成后，**保留**模板中的 `<!-- ⤷ 3c 回填 -->` 标记行——它们为步骤 3c 提供回填定位指引，届时一并删除。

#### 步骤 3b：对照研究纲领四维比对

在上述填充完成后，读取 `../研究纲领.md`（vault 根）。对全文内容进行四维比对，生成 `{relevance_judgment}` 内部变量：

| 比对维度 | 问题 | 输出 |
| --- | --- | --- |
| 研究对象/体系 | 论文研究的材料、器件、工况是否在研究纲领覆盖范围内？ | 具体交集或差异 |
| 方法可迁移性 | 论文使用的实验/仿真/理论方法是否在研究纲领的工具箱中或可补充？ | 可迁移或不可用 |
| 结论相关性 | 论文结论是否支撑、挑战或补充了纲领中"当前关注的问题"？ | 具体的支撑/挑战/无关 |
| 知识空白 | 论文是否填补了纲领中列出的某个知识缺口？ | 填补或未覆盖 |

生成的 `{relevance_judgment}` 须包含以上四个维度的具体判断（而非泛泛的"相关"）。

#### 步骤 3c：回填研究纲领依赖字段

基于 `{relevance_judgment}`，回填模板中所有 `<!-- ⤷ 3c 回填 -->` 标记的字段。按标记载明的粒度填入。回填完成后，删除模板中的所有 `<!-- ⤷ 3c 回填 -->` 标记行——这些行不应出现在 wiki 页面中。

### 步骤 4：知识网络化（概念页面）

对于步骤 2 提取的每个概念：

**目标目录：** `wiki/concepts/`

**处理逻辑：**
0. **去重检查**：浏览 `wiki/concepts/` 下已有页面标题，对每个标题相似或疑似语义等价的已有页面，读取其正文定义，判断是否存在语义等价但命名不同的页面。若存在 → 合并到已有页面，将当前论文使用的别名追加到该页面的 `aliases` 字段；否则继续步骤 1。

   > 去重合并指引（缩写↔全称、近义词、上下位关系）见 `.claude/skills/ingest/概念页面模板.md` 开头区域。
1. 页面不存在 → 读取 `.claude/skills/ingest/概念页面模板.md`，按模板创建
2. 页面已存在 → 读取现有内容，**增量合并**新信息
3. **发现冲突** → **立即暂停**，向用户报告冲突内容，询问处理方式后再继续

### 步骤 5：创建方法页面（按需触发）

方法页面不再每篇论文强制创建，改为满足以下条件之一时触发：

1. **累积阈值**：同一方法在 ≥ 3 篇论文的「涉及的表征方法」区域中出现，值得汇总统编
2. **核心方法**：方法本身是研究纲领工具箱中的关键表征手段（如 LFA、X-ray CT、SEM、红外热像等）
3. **用户指定**：用户处理某篇论文时主动要求为某方法建页

**未触发时**：方法信息仅保留在各精读笔记的「涉及的表征方法」区域中。

**触发后**：将该方法在已处理论文中的所有出现汇总，在 `wiki/methods/` 中创建或更新独立页面。

**处理逻辑**（同步骤 4）：
1. 扫描 `wiki/sources/` 下所有精读笔记的「涉及的表征方法」区域，提取该方法的全部出现
2. 页面不存在 → 读取 `.claude/skills/ingest/方法页面模板.md`，按模板创建
3. 页面已存在 → 读取现有内容，**增量追加**新论文中的证据条目
4. 发现冲突 → 暂停并报告

### 步骤 6：更新全局注册表

**更新 vault 根 `../index.md`（统一索引）：**
每次 ingest 后更新 vault 根 `../index.md`。先读取当前 `../index.md`，找到 `## {类别}` 下当前子库的 `### {子库名}` 分组，在该分组内新增或更新条目。条目标题使用完整路径 `[[{子库名}/wiki/{分类}/{页面名}]]`。若该子库在对应类别下尚无分组，则新建 `### {子库名}` 子标题。保留已有条目不变，按年份排序插入新条目。
- Sources: 格式为 `[[{子库名}/wiki/sources/{年份} - {标题}]] — 该论文的核心主旨`，按年份排列。
- Concepts: 格式为 `[[{子库名}/wiki/concepts/{概念名}]] — 该概念的核心定义`。
- Methods: 格式为 `[[{子库名}/wiki/methods/{方法名}]] — 该方法测量的物理量及用途`。

**更新 `wiki/log.md`：**

追加操作日志（Append-only）。若 `wiki/log.md` 不存在则新建：

```markdown
## [YYYY-MM-DD] ingest | 操作简述
- **变更**: 新增 [[PageName]]; 更新 [[../index|统一索引]]
- **冲突**: 无 (或: 冲突 [[ConflictingPage]], 已暂停等待决策)
```

### 步骤 7：归档前的强制自检

**进入归档操作之前，必须完成以下自检，不得跳过：**

1. **核对阅读确认清单**：所有章节均为 ✓，且清单已向用户输出（不得内部跳过）。
2. **核对模板字段**：frontmatter 及所有正文区域已填入实质性内容，无"待填充""暂无"等敷衍占位。
3. **核对纲领依赖字段已回填**：模板中所有 `<!-- ⤷ 3c 回填 -->` 标记的字段已填入实质性内容，含指定的粒度要求。
4. **禁止修改源文件**：上述检查通过后方可执行归档操作，归档时绝对禁止修改 raw/ 源文件内部文字。

以上四项全部通过后，方可执行以下归档操作：

将源文件移动到归档目录。**注意区分文件类型**：

| 源文件类型 | 归档路径 |
|-----------|---------|
| `raw/papers/*`（普通文件，非论文子目录） | `raw/archive/` |
| `raw/papers/{年份 - 标题}/`（论文子目录） | `raw/archive/{年份 - 标题}/`（整体搬移，不含原始 PDF） |

> **论文子目录整体归档**：每次归档移动整个 `{年份 - 标题}/` 目录到对应归档路径。目录中所有文件——主文献 Markdown、支撑材料 Markdown（若有）、`_zotero_meta.json`——作为整体一并归档，不拆分、不单独清理。原始 PDF 已在 mineru-convert 阶段删除，无需处理。

**绝对禁止修改源文件内部的文字。**

归档完成后，清理 `raw/papers/` 中因移走论文子目录而产生的空目录（保留根目录 `raw/papers/`）。

**归档后路径回写**：归档完成后，将**本次 ingest 涉及的 wiki 页面**中指向本次归档论文的 `raw/papers/` 路径更新为 `raw/archive/`：
- source 页面：`sources_main` 和 `sources_supplementary`（若有）中指向本次归档论文的路径
- concept 和 method 页面：`sources` 列表中指向本次归档论文的条目
**注意**：仅替换**指向本次归档论文**的路径，不得替换指向仍在 `raw/papers/` 中其他论文的路径。同时检查 source 页面正文（如来源追踪表格）和 concept 页面正文中是否有指向本次归档论文的 `raw/papers/` 路径，若有同样替换。其余不变。

### 步骤 8：单篇完成确认

每篇论文的完整处理（步骤 1–7）完成后，输出以下信息并等待用户确认：

## 冲突处理流程

当发现新旧知识冲突时：

1. **暂停**：停止当前 ingest 流程
2. **报告**：向用户说明冲突内容（哪个页面、冲突点是什么）
3. **询问**：请用户选择处理方式：
   - A) 保留新旧两者，标注为"知识冲突"
   - B) 用新知识覆盖旧知识
   - C) 放弃本次 ingest
4. **继续**：根据用户选择继续或终止

# new-sublib 技能

从 `_子库模板/` 复制骨架，替换占位符，快速创建空白子库。

## 触发逻辑

1. **用户执行 `/new-sublib <子库名>`**：确认名称和描述后创建
2. **用户执行 `/new-sublib`**：询问子库名和研究领域描述后创建
3. **隐式触发**：用户说"创建新子库"、"新建一个子库"、"初始化子库"

---

## 步骤 1：确认子库信息

**子库名**：若用户已提供（如 `/new-sublib 电子散热`），使用该名称并清理非法字符。否则询问名称。

**研究领域描述**：若用户未在首次输入中同时提供描述，追问：

> 这个子库的研究领域是什么？（一句话描述）

若用户未提供描述或明确表示跳过，默认使用子库名作为研究领域描述。

子库名同时是：
- **目录名**（文件系统路径）

**命名约束**：仅含中文、英文、数字、连字符和下划线，不含路径分隔符（`/\`），不超过 64 字符。若用户输入的名称违反约束，自动清理（非法字符替换为 `-`）。

---

## 步骤 2：检查是否已存在

```bash
test -d "E:/Knowledge/{子库名}" && echo "EXISTS" || echo "NOT_EXISTS"
```

若目录已存在，报错：

> ❌ 子库 "{子库名}" 已存在。如需重建，请先手动删除或重命名现有目录。

停止执行。

---

## 步骤 3：复制模板

```bash
cp -r "E:/Knowledge/_子库模板/" "E:/Knowledge/{子库名}/"
```

---

## 步骤 4：替换占位符

对以下文件执行占位符替换：

| 文件 | 占位符 | 替换为 |
|------|--------|--------|
| `{子库名}/CLAUDE.md` | `{{子库名}}` | `{子库名}` |
| `{子库名}/CLAUDE.md` | `{{一句话描述}}` | 用户提供的研究领域描述 |
| `{子库名}/wiki/log.md` | `{{创建日期}}` | 当前日期 `YYYY-MM-DD` |

使用 `sed` 执行替换。先对变量中的 sed 特殊字符（`&`、`\`、`|`）做转义，再执行替换：

```bash
SUB="新子库名"
DESC="研究领域描述"
TODAY=$(date +%F)

# 转义替换串中的特殊字符
DESC_ESC=$(echo "$DESC" | sed 's/[&\\|]/\\&/g')
SUB_ESC=$(echo "$SUB" | sed 's/[&\\|]/\\&/g')

# CLAUDE.md
sed -i "s|{{子库名}}|${SUB_ESC}|g; s|{{一句话描述}}|${DESC_ESC}|g" "E:/Knowledge/${SUB}/CLAUDE.md"

# wiki/log.md
sed -i "s|{{创建日期}}|${TODAY}|g" "E:/Knowledge/${SUB}/wiki/log.md"
```

> `sed -i` 在 GNU sed（Git Bash 默认）下无需额外参数；若在 macOS 上运行，需改为 `sed -i ''`。

替换完成后，验证无残留占位符：

```bash
# 检查是否还有未替换的 {{...}} 占位符
grep -rn '{{.*}}' "E:/Knowledge/${SUB}/" 2>/dev/null
```

若有输出 → 报错，列出残留的占位符和所在文件，不继续后续步骤。

---

## 步骤 5：输出创建简报

```
✅ 子库创建完成：「{子库名}」

目录结构：
  {子库名}/
  ├── CLAUDE.md
  ├── raw/
  │   ├── papers/
  │   └── archive/
  ├── wiki/
  │   ├── log.md
  │   ├── concepts/
  │   ├── methods/
  │   ├── sources/
  │   └── syntheses/
  └── assets/

下一步：
  cd {子库名}
  开始使用 /zotero-import、/ingest 等命令
```

---

## 硬约束

- **不覆盖已有子库**：目录已存在则报错，不执行任何写操作
- **占位符全部替换**：创建完成后模板中不能残留 `{{...}}` 占位符

# query 技能

## 核心目标
将用户的提问转化为对本地 Wiki 的深度检索。提取相关页面信息，综合出带有明确引用来源的双链回答。当回答具有高价值时，主动将其固化为知识库的一部分。

## 触发场景
- 用户输入 `/query <问题>`
- 用户用自然语言询问：`"我的笔记里关于 X 是怎么说的"`、`"过去我对 Y 的决策是什么"`、`"查询 Z 相关的知识"`
- 用户提及 wiki、知识库、笔记、记录等关键词

## 检索与综合流水线

### 步骤 0：检索层级判定

按以下优先级逐级降级检索：

```
wiki/sources/（精读页面）
  → 未命中 → raw/archive/（已归档论文原文）
    → 未命中 → 通用知识
```

### 步骤 0.5：单页面快速通道

**触发条件**：用户提问中**明确提到了某篇论文的标题关键词、某个概念的完整名称或某个 wiki 页面名**（而非模糊的"我关于 X 的笔记"）。信号包括：
- 包含完整论文标题或标题中的独特关键词（如"van Erp 2020 的微通道宽度"、"Co-designing electronics with microfluidics 那篇"）
- 包含明确的 wiki 概念名（如"歧管微通道的 COP 是多少"）
- 包含年份+主题的组合（如"2023 年的 PCM 综述里怎么说的"）

**判定与执行逻辑**：

1. **直接匹配 wiki 页面**：将用户问题中的关键词与全库 wiki 页面进行匹配。用 Glob 搜索，**path 参数设为 vault 根目录（`../`）**，pattern 为 `*/wiki/sources/`、`*/wiki/concepts/`、`*/wiki/methods/`、`syntheses/`（跨子库搜索）。
   - 若**唯一匹配**：直接读取该页面 → 若页面内容足够回答问题，进入步骤 5 综合回答；若信息不足，进入 2
   - 若**多个匹配**：列出匹配项，进入步骤 1 通过索引进一步定位
   - 若**无匹配**：回退至步骤 1 正常索引流程

2. **沿 sources_main 回退到归档原文**：若 wiki 页面存在但信息不足以回答问题，读取该页面的 YAML frontmatter 中的 `sources_main` 字段，直接打开对应路径的原始论文 Markdown（若为相对路径以子库名开头，拼接 vault 根目录；若路径含 `raw/papers/` 但文件不存在，替换为 `raw/archive/` 后重试）。从原文中搜索相关信息后回答。

> **注意**：此步骤是步骤 1 的**前置加速器**——仅在用户明确指向单篇内容时触发。若用户问题涉及跨页面比较、领域综述、或多个概念，跳过此步骤直接进入步骤 1。

### 步骤 1：查阅全局统一索引
**若步骤 0.5 已命中并充分回答，跳过此步骤**。否则读取 vault 根 `../index.md`（统一索引）

在统一索引中定位与问题相关的页面，跨所有子库：
- Concepts（概念）
- Methods（方法）
- Sources（来源）
- Syntheses（综合）

### 步骤 2：第一级 — 深度阅读 wiki 页面
逐一读取步骤 1 中找到的所有相关页面。不限于单个页面——若问题涉及多个概念或交叉领域，须全部读完再综合。

若相关页面提供了足够信息 → 直接进入步骤 5 综合回答。若信息不足 → 继续步骤 3。

### 步骤 3：第二级 — 回退到归档论文原文
若步骤 1-2 在 wiki 中未找到相关内容：

**优先路径 — 通过 sources_main 直接定位**：若步骤 2 中已读取了 wiki source 页面（精读笔记）但信息不足，则先读取该页面的 YAML frontmatter，提取 `sources_main` 字段，直接打开该路径指向的原始论文 Markdown。若 `sources_main` 为相对路径（以子库名开头），拼接 vault 根目录得到绝对路径。若文件不存在（旧版 ingest 可能未回写路径，`sources_main` 仍指向已搬移到 `raw/archive/` 的旧 `raw/papers/` 位置），将路径中的 `raw/papers/` 替换为 `raw/archive/` 后重试。

**降级路径 — 关键词扫描**：若以上优先路径未命中（wiki source 页面无 `sources_main` 或路径指向的文件不存在），则：
1. 扫描 `raw/archive/` 下的论文子目录，按问题关键词匹配目录名（论文标题）
2. 对匹配到的论文，读取其 Markdown 原文（`{年份} - {标题}.md`）
3. 从原文中搜索相关信息片段
4. 若找到 → 以该论文原文为依据回答，引用时标注 `（来自归档原文：{年份} - {标题}）`

> **注意**：此步骤仅在 wiki 精读页面无相关内容时触发。若 wiki 已有答案，不回溯 archive。

### 步骤 4：第三级 — 降级到通用知识
若步骤 1-3 在 wiki 和 raw/archive/ 中均未找到相关内容：

> 本地知识库中未找到相关内容，以下为通用知识回答：[直接回答]

### 步骤 5：综合与回答
综合信息，回答用户问题。

**双链引用规范**：
- 每当引用某个 Wiki 页面的信息，在文本中使用 `[[页面名称]]` 标注
- 整段引用同一页面：段落首尾各引用一次
- 引用特定原文：使用 Markdown 块引用 `> 引用内容`
- 引用归档原文时标注 `（来自归档原文：{年份} - {标题}）`

### 步骤 6：高价值内容固化
如果满足以下条件，主动询问用户是否保存为 synthesis：
- 回答超过 2 个段落
- 内容具有分析对比性或总结性

询问话术：
>这是一个有价值的总结，是否需要我将其保存到 syntheses/ 目录？

用户同意后，按照以下规则创建文件：

**文件名**：使用简体中文 TitleCase 命名，概括综合主题（如 `液冷与微通道散热对比分析.md`）。若综合内容围绕特定论文展开，则沿用 `{年份} - {标题}.md` 格式。

**保存路径**：`syntheses/{文件名}.md`

**模板**：
```markdown
---
title: "{{综合主题}}"
aliases: []
type: synthesis
tags:
  - {{标签}}
sources:
  - [[本次查询阅读的 wiki 页面双链]]
created: YYYY-MM-DD
---

# {{综合主题}}
```

> **`sources` 字段**：填入本次查询中实际阅读并引用的 wiki 页面双链（非 raw/ 路径），每行一个 `[[页面名]]`。

并在 vault 根 `../index.md`（统一索引）的 Syntheses 分类下注册。

### 步骤 7：记录操作日志
无论是否生成 synthesis 页面，查询结束后必须在 `wiki/log.md` 末尾追加：

```markdown
## [YYYY-MM-DD] query | <操作简述>
- **输出**: <引用页面列表或"即时回答未保存">
```

格式必须完全遵循 CLAUDE.md 中的示例。

## 强制约束
- **禁止凭记忆回答**：必须通过检索 wiki 页面或归档原文获取答案来源（经由步骤 0.5 快速通道或步骤 1-3 正常流程），不得仅依赖模型自有知识
- **禁止过度引用**：同一页面的信息在段落首尾引用一次即可
- **禁止静默回答**：知识库无相关内容时必须声明
- **禁止跳过 archive**：wiki 无相关内容时，必须先检索 raw/archive/ 再降级到通用知识

# 研究日志技能

逐类询问用户今天的研究活动，自动搜索 wiki 已有页面生成双链引用，整合后写入 `研究日志/YYYY-MM.md`（按月分文件，统一存放在 vault 根下的 `研究日志/` 目录）。

## 触发逻辑

1. **用户执行 `/research-log`**：进入逐类问答模式
2. **用户执行 `/research-log 快速`**：跳过逐类问答，一次收集所有活动后整合
3. **用户执行 `/research-log <日期>`** 或 `/research-log 昨天`：为过去某一天补充记录，逻辑相同
4. **隐式触发**：用户说"记录今天的研究"、"写研究日志"、"今天做了什么"

---

## 步骤 0：识别当前子库

按以下优先级推断用户正在哪个子库下工作：

**1. `pwd` 检测**：
- 若 `pwd` 为 `E:\Knowledge\{子库名}`（如 `E:\Knowledge\电力电子综述`）→ 使用目录名作为标识符
- 若 `pwd` 为 `E:\Knowledge`（vault 根）→ 进入下一级 fallback

**2. Fallback：上下文文件路径推断**（当 `pwd` 为根时）：
- 检查当前会话中已打开或操作过的文件路径（如最近 Read/Edit 的文件、当前 ingest 的目标文件）
- 若路径匹配 `{子库}/wiki/`、`{子库}/raw/` 等子库内部结构，提取 `{子库名}` 作为标识符

**3. Fallback：用户确认**（前两级都无结果时）：
- 列出 `E:\Knowledge` 下已有的子库目录（排除以 `_` 和 `.` 开头的目录）
- 询问用户记录到哪个子库，支持手动输入新子库名

子库标识符（目录名）用于在日志条目中标注活动归属：

```markdown
### 2026-05-29 (周四)
**电力电子综述**：
- **阅读文献**：...
```

同一子库同一天有多条活动时，在其子库小节下追加。跨子库活动时，分别建立子库小节。

---

## 步骤 1：确定日期

从用户输入中解析目标日期（默认今天）：

| 用户输入 | 解析结果 |
| --- | --- |
|（无日期参数）| 当天日期 `2026-05-27` |
| `昨天` | 当天日期减 1 天 |
| `前天` | 当天日期减 2 天 |
| `YYYY-MM-DD` | 直接使用 |
| `周一`/`周二`/... | 本周最近的那个周几 |

**当天日期的获取**：当前日期为 {{currentDate}}。

---

## 步骤 2：逐类询问活动

按以下顺序逐一询问，每轮只问一个分类。用户回答"跳过"、"无"、"没有"、"没"、"none"、"no"时跳过该分类。

**提问规范**：
- 每轮提问前保持短小，一行即可
- 如果用户回答中包含论文标题、概念名或方法名，启动步骤 3 的双链搜索
- 每轮将用户回答存入临时变量，不立即写入文件

### 2.1 活动分类与提问语

| # | 分类 | 提问 |
|---|------|------|
| 1 | **阅读文献** | 今天读了哪些论文？有什么收获？ |
| 2 | **检索文献** | 有没有检索文献？搜了什么主题或数据库？ |
| 3 | **仿真实验** | 今天做仿真了吗？什么内容、进展如何？ |
| 4 | **实验制备** | 有没有制备样品？做了什么、有调整工艺吗？ |
| 5 | **测试表征** | 做了哪些测试或表征？结果怎么样？ |
| 6 | **数据分析** | 分析数据了吗？有什么发现？ |
| 7 | **论文写作** | 写论文了吗？写了哪部分、进度如何？ |
| 8 | **会议讨论** | 有参加组会或讨论吗？关键结论是什么？ |
| 9 | **其他** | 还有其他研究相关的工作吗？ |

### 2.2 快速模式（`/research-log 快速`）

跳过逐类问答，一次性提示用户自由输入：

> 今天做了哪些研究相关的事？（可以按分类列出：阅读文献、仿真实验、测试表征、数据分析、论文写作、会议讨论等）

用户自由输入后直接进入步骤 3，对输入内容中可识别的论文/概念/方法进行双链搜索。

---

## 步骤 3：双链补全

### 3.1 识别候选实体

从用户回答中识别可能需要双链的实体：

- **论文标题**：完整标题或明显的论文描述（如"那篇关于微流控散热的综述"）
- **概念/术语**：已在 `wiki/concepts/` 中有页面的专业名词
- **方法/工具**：已在 `wiki/methods/` 中有页面的方法名
- **研究纲领相关**：与研究领域、关键物理量、当前问题相关的表述 → 链接到 `[[研究纲领]]`

### 3.2 搜索匹配

对每个候选实体，执行搜索：

1. **论文**：在 `wiki/sources/` 中根据标题关键词匹配文件名。若有已摄入的精读页面，用 `[[{年份} - {标题}]]` 引用；若未摄入但用户提及，标注 `（待摄取）`
2. **概念/方法**：列出 `wiki/concepts/` 和 `wiki/methods/` 已有的页面标题，匹配用户回答中的术语
3. **研究纲领**：用户提到的研究方向、物理量、瓶颈问题等，若在研究纲领中有对应条目，链接到 `[[研究纲领]]`

### 3.3 匹配不到

匹配不到的论文/概念/方法，在日志条目后追加 `（待摄取）` 提示：

```markdown
- 读了 Chen 等 2025 关于歧管微通道优化设计的论文（待摄取）
```

### 3.4 注意

- **只链接已有 wiki 页面的实体**，不要为不存在的页面创建空双链
- 用户回答中提到的文件名或路径需要判断是否为已有资源

---

## 步骤 4：整合预览

### 4.1 组装日志正文

将步骤 2 收集到的各分类回答整合为一个 Markdown 条目。

**组装规则**：
- 仅包含用户有实质内容的分类，空白/跳过分类不出现在条目中
- 条目外层包裹 `**{子库名}**：` 标注活动归属，其下每个有内容的分类成为子条目，格式：`- **分类名**：\n  - 具体内容`
- 子条目中已识别的实体替换为双链
- 保留用户原意，整理为简洁条目（去除口语填充词，保留实质性信息）

### 4.2 日期条目结构

```markdown
### YYYY-MM-DD (周X)
**{子库名}**：
- **阅读文献**：
  - [[{年份} - {标题}]] — 核心收获一句话
- **仿真实验**：
  - 具体进展内容
```

同一子库同一天追加时，在已有子库小节下直接追加子条目，不重复子库标题。跨子库时新增子库小节。

### 4.3 预览并确认

将组装好的 Markdown 展示给用户预览，格式如下：

```
📋 以下内容将写入 研究日志/YYYY-MM.md（标注双链的条目已自动匹配）：

### 2026-05-27 (周三)
**电力电子综述**：
- **阅读文献**：
  - ...
- **仿真实验**：
  - ...

是否写入？(y/n)
```

- 用户确认 `y`/`是`/`好`/`可以`/`yes` → 进入步骤 5 写入
- 用户拒绝 `n`/`否`/`不`/`别`/`no` → 询问是否需要修改：修改哪部分，直接在对话中告知
- 修改后再次预览确认

---

## 步骤 5：写入研究日志

### 5.1 定位月度文件

根据步骤 1 确定的目标日期，构造文件路径 `研究日志/YYYY-MM.md`（其中 `MM` 为目标月份，位于 vault 根）。

1. 检查该文件是否存在
2. **若存在** → 直接读取
3. **若不存在** → 新建文件，写入以下骨架：

```markdown
---
title: "研究日志 YYYY-MM"
type: log
tags:
  - meta
  - research-log
created: YYYY-MM-01
---

# 研究日志 YYYY年MM月

## [YYYY-Www] MM.DD - MM.DD（{{本周核心进展，周末填写}}）

## 关联连接

- [[index|研究日志索引]]
- [[../index|Knowledge Vault]]
```

日条目内容由后续步骤 5.2-5.3 负责写入，骨架不预置日条目占位符。

同时更新 `研究日志/index.md`，在"月度日志"区块追加 `- [[YYYY-MM]] — YYYY年MM月`。

### 5.2 定位写入位置

读取月份文件后：
- 若当天日期已有条目 → 在已有 `### YYYY-MM-DD (周X)` 标题下查找对应子库小节；若已有该子库小节，在其下追加；否则在当天条目末新增子库小节
- 若当天日期无条目 → 在所在周的 `## [YYYY-Www]` 标题下新增 `### YYYY-MM-DD (周X)` 条目，其下新建子库小节
- 若当前周标题不存在 → 新建周标题 `## [YYYY-Www] MM.DD - MM.DD（{{本周核心进展，周末填写}}）`

### 5.3 写入格式

当天日期无条目时完整写入：

```markdown
### 2026-05-29 (周四)
**{子库名}**：
- **阅读文献**：
  - [[{年份} - {标题}]] — 核心收获一句话
- **仿真实验**：
  - 具体进展内容
```

当天已有该子库条目时追加示例：

```markdown
### 2026-05-27 (周三)
**电力电子综述**：
- **阅读文献**：
  - [[2025 - Direct-to-Package Cooling for AI Chips]] — 微流控歧管设计的压降权衡有新思路
- **仿真实验**：
  - 完成了微通道散热器的参数化扫描，热阻最低降至 0.12 K/W ← 新追加
```

### 5.4 更新研究日志操作日志

在子库的 `wiki/log.md` 追加：

```markdown
## [YYYY-MM-DD] research-log | 记录研究日志
- **变更**: 更新 [[../../研究日志/YYYY-MM|研究日志]]
- **活动**: 阅读文献、仿真实验、...
```

---

## 注意事项

- 不要为不存在的页面创建空双链——只链接已有 wiki 页面
- 研究宣言/目标/方向类的内容链接到 `[[研究纲领]]`，不链到其他具体页面
- 使用简体中文编写所有内容
- 保持条目简洁，去除口语填充词（"嗯"、"然后"、"就是"等）
- 不要虚构或猜测用户没提到的活动内容
- 只收录研究相关的活动，日常生活（吃饭、通勤等）不记录

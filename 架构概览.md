# Knowledge Vault 架构概览

## 一、目录结构

```
Knowledge/                          ← Vault 根
├── CLAUDE.md                       ← 全局规范（语言、架构、权限、YAML 契约）
├── index.md                        ← 全库统一索引
├── log.md                          ← 全局操作日志
├── 架构概览.md                     ← 本文件
├── 研究纲领.md                     ← 全体子库共用研究纲领
├── syntheses/                       ← 跨子库综合分析（vault 根，扁平结构）
├── 研究日志/                       ← 统一研究日志（按月分文件）
│   ├── index.md
│   └── YYYY-MM.md
├── .claude/skills/                 ← 全局共享 Skills（12 个自定义）
│   ├── zotero-import/
│   ├── mineru-convert/
│   ├── ingest/
│   ├── query/
│   ├── lint/
│   ├── research-log/
│   ├── new-sublib/
│   ├── defuddle/
│   ├── json-canvas/
│   ├── obsidian-bases/
│   ├── obsidian-cli/
│   └── obsidian-markdown/
├── _子库模板/                      ← 新建子库时复制
└── 电力电子综述/                     ← 子库实例
    ├── CLAUDE.md                   ← 子库说明
    ├── raw/                        ← 不可变层
    │   ├── papers/{年份 - 标题}/   ← 待处理论文
    │   └── archive/                ← 已归档
    ├── wiki/                       ← 编译输出层
    │   ├── log.md                  ← 子库操作日志
    │   ├── concepts/               ← 概念页（TitleCase 命名）
    │   ├── methods/                ← 测试/表征/数据处理方法
    │   └── sources/                ← 论文精读（{年份} - {标题} 命名）
    └── assets/                     ← 媒体资产
```

### 三层权限模型

| 层 | 目录 | 权限 |
|----|------|------|
| 不可变层 | `raw/` | 只读，禁止修改 |
| 媒体层 | `assets/` | 存放图片，wikilink 引用 |
| 编译层 | `wiki/` | 自由创建、更新、提炼 |

### 跨层引用规则

| 引用类型 | 语法 | 示例 |
|---------|------|------|
| 同子库内 | `[[页面名]]` | `[[微通道散热]]` |
| 跨子库 | `[[子库名/wiki/.../页面名]]` | `[[电力电子综述/wiki/concepts/微通道散热]]` |
| 子库→vault 根 | `[[../文件名|显示名]]` | `[[../研究纲领|研究纲领]]` |

## 二、12 个自定义 Skill

### 论文导入流水线（3 个）

```
Zotero ──→ raw/papers/ ──→ wiki/ ──→ raw/archive/
  ↑            ↑              ↑           ↑
zotero-import  mineru-convert  ingest    (自动归档)
```

| Skill | 输入 | 输出 | 职责 |
|-------|------|------|------|
| **zotero-import** | Zotero 分类名 | `raw/papers/{年份-标题}/` 含 PDF + `_zotero_meta.json` | 从数据库拷贝 PDF，去重，分批导入 |
| **mineru-convert** | `raw/papers/` 中的 PDF | 同级 `.md` + `assets/` 图片 | VLM 引擎 PDF→Markdown，图片路径修正为 vault 级 wikilink |
| **ingest** | `raw/papers/` 中的 `.md` | `wiki/sources/`、`wiki/concepts/`、`wiki/methods/` | 提炼主旨、综述检测、四维比对研究纲领、创建概念/方法页、更新索引、归档 |

**ingest 核心逻辑**：

- **研究论文** → 模板 A（含方法页创建）+ 对照纲领四维比对
- **综述论文** → 模板 B（叙事逻辑拆解 + 批判性评论 + 研究方向推荐）
- **概念提取** → 去重合并（别名匹配、同义词判定、上下位关系处理）
- **综述检测** → 5 信号综合判定（标题关键词 + 摘要特征 + 文章类型 + 结构特征 + 路径兜底）

### 知识维护（4 个）

| Skill | 职责 | 核心功能 |
|-------|------|---------|
| **new-sublib** | 子库创建 | 复制模板、替换占位符 |
| **lint** | 知识库健康巡检 | 8 步检查：索引一致性 → 死链/孤儿 → 关联连接 → 知识冲突 → 概念/方法去重 → 公式规范性 → 收件箱积压 → 纲领同步 |
| **query** | 检索与问答 | 三级检索降级（wiki → archive → 通用知识）→ 双链引用回答 → 高价值内容固化为 synthesis |
| **research-log** | 研究日志记录 | 9 类逐类问答 → 双链补全 → 预览确认 → 写入 `研究日志/YYYY-MM.md` |

### Obsidian 创作工具（4 个）

| Skill | 职责 | 核心功能 |
|-------|------|---------|
| **obsidian-markdown** | Obsidian Markdown 创作 | wikilink、embed、callout、frontmatter、tag 等 Obsidian 特有语法 |
| **obsidian-bases** | .base 文件管理 | 创建/编辑 Obsidian Bases（视图、筛选、公式、汇总） |
| **json-canvas** | .canvas 文件管理 | 创建/编辑 JSON Canvas（节点、边、分组） |
| **obsidian-cli** | Obsidian CLI 交互 | 通过命令行读写笔记、搜索、管理任务、插件开发调试 |

### 外部内容获取（1 个）

| Skill | 职责 | 核心功能 |
|-------|------|---------|
| **defuddle** | 网页内容提取 | 去除导航/广告/杂乱内容，提取干净 Markdown，节省 token |

**lint 的 8 步巡检**：

| 步骤 | 检查项 | 输出 |
|------|--------|------|
| 1 | 索引一致性 | 注册但文件缺失 / 文件存在但未注册 |
| 2 | 双向链接健康 | 死链（含 aliases 解析）+ 孤儿（含白名单） |
| 3 | 必填区域 | 缺失 `## 关联连接` 的页面 |
| 4 | 认知冲突 | 含 `## 知识冲突` 区块的页面 |
| 5 | 概念/方法语义去重 | 高/中/低置信度重复页面（含跨子库检测） |
| 6 | 数学公式规范性 | 7 类公式问题（未闭合 `$$`、Unicode 字符、下标变量等） |
| 7 | 收件箱积压 | 提示由 `/ingest` 负责 |
| 8 | 研究纲领同步 | 日志 vs 纲领对比 + 精读关联判断过时检测 |

**lint 报告四级输出**：

| 级别 | 含义 | 示例 |
|------|------|------|
| 绿灯 | 检查通过 | 所有页面均含关联连接、索引入口全部有效、纲领与研究一致 |
| 建议 | 信息提示 | 纲领关注点可细化 |
| 黄灯 | 需关注 | 孤儿页面、未同步索引、纲领覆盖缺口/冗余、精读关联判断过时 |
| 红灯 | 需立即处理 | 死链、未解决的知识冲突 |

## 三、核心工作流

### 流程 A：论文入库（zotero-import → mineru-convert → ingest）

```
用户: /zotero-import 微通道
  │
  ├─ 1. 定位 Zotero 数据库 → 递归查找分类 → 提取论文元数据
  ├─ 2. 去重（按 wiki/sources/ frontmatter 中的 zotero_item_id）
  ├─ 3. 拷贝 PDF 到 raw/papers/{年份 - 标题}/ + 写入 _zotero_meta.json
  ├─ 4. 元数据补全（已导入论文的新增分类归属）
  │
  ├─ 5. /mineru-convert → PDF 转 Markdown（VLM hybrid-auto-engine）
  │     └─ 图片路径修正为 ![[../../assets/{pdf_stem}/xxx.png]]
  │
  └─ 6. /ingest
        ├─ 读论文 md → 提炼主旨、翻译、公式规范化
        ├─ 综述检测（5 信号综合判定）
        ├─ 对照 ../研究纲领.md → 四维比对 → {relevance_judgment}
        ├─ 生成精读笔记（模板 A/B）→ wiki/sources/{年份} - {标题}.md
        ├─ 概念提取去重 → wiki/concepts/
        ├─ 方法提取（仅研究论文）→ wiki/methods/
        ├─ 更新 vault 根统一索引 + wiki/log.md
        └─ 归档到 raw/archive/
```

**ingest 四维比对框架**：

| 维度 | 问题 |
|------|------|
| 研究对象/体系 | 论文研究的材料、器件、工况是否在研究纲领覆盖范围内？ |
| 方法可迁移性 | 论文使用的方法是否在纲领工具箱中或可补充？ |
| 结论相关性 | 论文结论是否支撑、挑战或补充了纲领中"当前关注的问题"？ |
| 知识空白 | 论文是否填补了纲领中列出的某个知识缺口？ |

### 流程 B：研究记录与纲领演进（research-log → lint）

```
日常: /research-log
  └─ 逐类记录（阅读文献/仿真实验/测试表征等 9 类）
      → 自动搜索 wiki 双链补全
      → 写入 ../研究日志/YYYY-MM.md（按月文件，vault 根）

定期: /lint
  ├─ Step 1-7: 结构健康检查
  └─ Step 8: 纲领同步检查
        ├─ 读 ../研究纲领.md + 最近 2 月研究日志
        ├─ 语义对比 → 覆盖缺口 / 冗余 / 关注点细化
        │     ├─ 黄灯：纲领需更新
        │     └─ 建议：关注点可细化
        │
        ├─ 若纲领有变化 → 扫描 wiki/sources/ 精读页面
        │     └─ 检测关联判断过时（低估 / 过时 / 细化不足）
        │
        └─ 用户确认修复
              ├─ 更新 ../研究纲领.md
              └─ 重跑四维比对 → 更新精读页面关联判断 + 追加变更记录
```

### 流程 C：知识问答（query）— 三级检索降级

```
/query <问题>
  │
  ├─ 步骤 1: 读 vault 根统一索引定位相关页面（Concepts/Methods/Sources/Syntheses）
  ├─ 步骤 2: 第一级 — 深度阅读 wiki 页面
  │     ├─ 命中 → 步骤 5 综合回答
  │     └─ 未命中 ↓
  │
  ├─ 步骤 3: 第二级 — 回退到 raw/archive/ 归档论文原文
  │     ├─ 按关键词匹配论文目录名 → 读取 Markdown 原文 → 搜索相关片段
  │     ├─ 命中 → 步骤 5 综合回答（标注"来自归档原文：{年份} - {标题}"）
  │     └─ 未命中 ↓
  │
  ├─ 步骤 4: 第三级 — 降级到通用知识
  │     └─ "本地知识库中未找到相关内容，以下为通用知识回答"
  │
  ├─ 步骤 5: 综合与回答（[[双链]] 引用）
  ├─ 步骤 6: 高价值内容固化（> 2 段 + 分析对比性 → syntheses/）
  └─ 步骤 7: 记录操作日志
```

> **archive 读取权限**：ingest 禁止读取 archive（防止重复摄入），但 query 可在 wiki 未命中时读取 archive 作为检索源。

## 四、数据流与关键契约

### 数据生产者/消费者

| 数据 | 生产者 | 消费者 | 载体 |
|------|--------|--------|------|
| Zotero 元数据 | zotero-import | ingest | `_zotero_meta.json`（归档时一并搬移） |
| 论文 Markdown | mineru-convert | ingest | `raw/papers/{年份-标题}/` |
| 研究纲领 | 用户手动维护 | ingest（四维比对）、lint（同步检查） | `../研究纲领.md` |
| 研究日志 | research-log | lint（纲领同步） | `../研究日志/YYYY-MM.md` |
| 精读关联判断 | ingest | lint（过时检测+修复） | `wiki/sources/{年份} - {标题}.md` |
| YAML aliases | ingest（概念创建） | lint（死链检测） | 各页面 frontmatter |
| 归档论文原文 | ingest（归档写入） | query（二级检索） | `raw/archive/{年份-标题}/` |
| 子库索引 | ingest / query | query / lint | vault 根 `index.md`（统一索引） |

### YAML Frontmatter 红线

- tags 必须多行列表格式，禁止内联数组 `[a, b]`
- 所有字符串值必须双引号包裹
- `sources_main` / `sources_supplementary` 路径必须含 `{{子库名}}/` 前缀（vault 级绝对路径），且必须单行
- 所有 wiki 页面必含 `aliases: []` 和 `created` 字段

### 页面命名规范

| 分类 | 命名规则 | 示例 |
|------|---------|------|
| Concepts | 简体中文 TitleCase | `微通道散热.md` |
| Methods | 简体中文 TitleCase | `激光闪光法.md` |
| Sources | `{年份} - {标题}` | `2025 - Direct-to-Package Cooling for AI Chips.md` |
| Syntheses | 论文相关用年份格式，否则 TitleCase（位于 vault 根 `syntheses/`） | `液冷与微通道散热对比分析.md` |

### 必填区域

每个 wiki 页面必须包含 `## 关联连接` 区域，使用 Obsidian 双链确保无孤岛页面。

## 五、子库模板初始化

新建子库时从 `_子库模板/` 复制：

```
_子库模板/
├── CLAUDE.md           ← 子库说明（含路径约定和关键文件双链）
├── raw/papers/         ← 空目录，待存放论文
├── raw/archive/        ← 空归档目录
├── wiki/
│   ├── log.md          ← 空操作日志
│   ├── concepts/       ← 空概念目录
│   ├── methods/        ← 空方法目录
│   └── sources/        ← 空精读目录
└── assets/             ← 空媒体目录
```

复制后需：编辑 `CLAUDE.md`，设置标题和描述。
